# Mobile Header Design

**Date:** 2026-04-04

## Goal

モバイルでヘッダーが横幅不足により崩れる問題を解消し、`ロゴ + ハンバーガー + 上部ドロワー` の構成で最小限かつ読みやすい導線に整理する。

## Current Problem

- モバイルでもデスクトップと同じテキストナビを横並びで表示している
- サイト名が長いため、ナビ文言が詰まりやすく、縦書きのような不自然な見え方が起きる
- テーマ切り替えが常設されており、狭い幅で優先度の低い操作が場所を取っている

## Chosen Direction

- モバイルではナビリンクの横並び表示をやめる
- 左側にフルテキストのロゴを残す
- 右側は軽量なハンバーガーアイコンだけを置く
- ハンバーガー押下でヘッダー直下に小さめのドロワーを表示する
- ドロワー内に `ホーム / ブログ / カテゴリ / About / テーマ切替` を配置する
- テーマ切替はモバイルではメニュー内に移し、常設しない

## UI Structure

### Desktop

- 既存に近い横並びナビを維持する
- テーマ切替は右端に残す
- モバイル用ドロワーは非表示

### Mobile

- ロゴはフルテキストのまま表示
- ハンバーガーボタンを右端に配置
- ドロワーは上から落ちる小型パネルとし、画面全体は覆わない
- 項目は縦並びで、押しやすい高さを確保する
- テーマ切替は最後の行に分離して置く

## Interaction

- ハンバーガーで開閉する
- メニューは軽い `fade + slight slide` 程度で表示する
- 開閉状態は見た目で明確にする
- キーボード操作と読み上げを意識して、トグルには適切なラベルを付ける

## Implementation Notes

- `Header.astro` を中心に実装する
- モバイルメニューは軽量性を優先し、複雑な state 管理は避ける
- 既存のテーマ切替スクリプトは複数ボタンに対応させる
- 見た目確認のためのプレースホルダーではなく、本番でそのまま成立する構造にする

## Success Criteria

- モバイルでヘッダー内の文字が縦に崩れない
- ナビゲーションが最小構成でも迷わず使える
- テーマ切替がモバイルで無理なく使える
- デスクトップの既存導線は大きく損なわない
