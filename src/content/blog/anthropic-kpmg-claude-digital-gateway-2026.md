KPMG と Anthropic は **2026年5月19日**、Claude を KPMG の中核業務と顧客向けプラットフォームに組み込むグローバル提携を発表した。KPMG の全世界 276,000 人超の従業員が Claude を利用できるようになり、税務・法務クライアント向けには KPMG Digital Gateway の中で Claude Cowork と Managed Agents を使う構成が示された。

これは「大手コンサルが AI チャットを導入した」という話ではない。KPMG は、税務知見、独自ツール、クライアントデータが集まる Digital Gateway の中に Claude を入れ、専門職と顧客が同じ業務面で AI エージェントを使う方向を打ち出している。以前取り上げた [PwC Claude 展開](/blog/pwc-anthropic-claude-code-cowork-2026/) や [Anthropic と NEC の戦略協業](/blog/anthropic-nec-ai-engineering-workforce-japan-2026/) と同じく、Claude は個人ツールから、教育、統制、顧客提供、業務再設計を含む企業導入モデルへ進んでいる。

日本の税理士法人、監査法人、法律事務所、SIer、コンサル、事業会社の AI 推進部門にとって重要なのは、Claude の採用社数ではない。AI エージェントを、税務・法務・PE・サイバーセキュリティのような責任が重い業務に入れるとき、どのプラットフォームに置き、誰が確認し、どの証跡を残すかという設計が前面に出てきたことだ。

## 事実: ClaudeがDigital Gatewayに入る

Anthropic の発表によると、KPMG は Claude を Digital Gateway に組み込む。Digital Gateway は、KPMG の専門職とクライアントが実際の業務で使うソフトウェア基盤であり、税務・法務向けの新機能から始まる。KPMG 側の発表では、この基盤は Microsoft Azure 上に構築され、KPMG の税務知見、独自ツール、クライアントデータを一つの環境で扱うと説明されている。

ここで注目すべきは、Claude Cowork と Managed Agents が「別のチャット画面」としてではなく、Digital Gateway の中に入る点だ。KPMG は、税制変更への対応を支援する AI エージェントの構築が、以前は複数ツールとチャット画面を行き来する作業だったのに対し、Digital Gateway 内で進められるようになると説明している。

全従業員への展開も大きい。KPMG の 276,000 人超のグローバル人材が Claude の機能へアクセスできるようになる。これは米国の Advisory、AI and Data Labs、社内支援チームでの 2 年間の利用を広げるものとされる。つまり、KPMG は新規 PoC ではなく、既存の社内利用を顧客向け基盤と全社展開へ拡大している。

また Anthropic は、KPMG をプライベートエクイティ向けの優先パートナーに位置づけた。KPMG は PE ポートフォリオ企業へ Claude と Anthropic のエージェントを導入する支援を担い、KPMG Blaze のような PE 向けオファリングでは Claude Code を組み込み、レガシー IT の近代化や AI 対応システムの開発を速める構想が示されている。

## 分析: 業務プラットフォーム内AIが本丸になる

ここからは分析だ。

今回の発表で最も重要なのは、AI エージェントの置き場所が変わっていることだ。これまで多くの企業導入は、既存業務の外側に AI チャットを置き、ユーザーが資料や文章をコピーして相談する形から始まった。しかし税務・法務・監査・PE・サイバーセキュリティのような業務では、コピーして相談するだけでは限界がある。必要なのは、業務データ、専門ツール、承認フロー、顧客とのやり取りがある場所に AI を置くことだ。

KPMG の Digital Gateway は、その方向を分かりやすく示している。AI が単に回答するのではなく、税務知識、クライアントデータ、社内ツールが集まる業務面で、エージェント機能を作り、使い、顧客に届ける。これは [Claude 金融エージェント](/blog/anthropic-claude-finance-agents-2026/) で見た業務テンプレートの流れともつながる。金融、税務、法務のような領域では、汎用チャットより、業務単位で入力、出力、レビュー、ログを決められる形が重要になる。

一方で、この形は責任も重くする。Digital Gateway にクライアントデータがあるなら、AI がどのデータを参照したのか、どの権限で動いたのか、生成物を誰が確認したのかを残さなければならない。AI が便利になるほど、誤った助言、古い税制解釈、機密情報の扱い、法域の違い、監査独立性のような問題も近づく。

KPMG は Trusted AI framework、サイバーセキュリティ、リスク、AI assurance を組み込むと説明している。これは宣伝文句として読むだけでは足りない。日本企業が同じ構造を採るなら、AI エージェントを使う前に、データ分類、利用目的、責任者、レビュー基準、ログ保管、顧客説明のルールを決める必要がある。

## 日本企業が読むべきポイント

日本の専門サービス企業や事業会社にとって、今回の発表は三つの意味を持つ。

第一に、AI 導入は「個人利用」から「業務提供モデル」へ移っている。税務、法務、監査、コンサル、システム開発では、AI を社内効率化だけに閉じるより、顧客提供価値の一部として組み込む動きが強くなる。KPMG は Digital Gateway という既存の顧客向け基盤に Claude を入れることで、AI をサービス提供面に置いた。

第二に、大規模展開には教育と人間の役割設計が必要になる。Anthropic の発表では、KPMG と UT Austin の研究に触れ、人間が AI の横で何をするかが価値を左右するという観点が出ている。これは [PwC の 3万人認定計画](/blog/pwc-anthropic-claude-code-cowork-2026/) と同じ論点だ。AI を配るだけでは、専門職の判断、業務フローの変更、出力評価、意思決定の責任は定まらない。

第三に、AI 導入支援市場の競争が変わる。Anthropic は KPMG を PE 向けの優先パートナーにした。これは [Anthropic 企業AIサービス会社](/blog/anthropic-enterprise-ai-services-company-2026/) で整理した導入支援市場の拡大と同じ方向にある。AI ベンダーはモデルを売るだけでなく、コンサル、SI、投資会社、業務プラットフォームを通じて、導入の現場へ深く入ろうとしている。

日本の SIer やコンサルにとっては、単に「Claude を扱えます」と言うだけでは弱い。業務テンプレート、評価データ、プロンプトやスキルの変更管理、顧客データの境界、監査ログ、教育コンテンツを持てるかが差になる。税務・法務・金融・公共のような領域では、モデルの賢さより、業務として説明できる導入設計のほうが契約の決め手になりやすい。

## 導入前に確認すべきこと

日本企業が KPMG 型の導入を参考にするなら、最初に確認すべきは AI を置く業務面だ。汎用チャットから始めるのか、社内ポータルに入れるのか、CRM、DMS、会計、契約管理、開発基盤の中に入れるのかで、統制の重さは変わる。業務データが集まる場所に入れるほど効果は大きいが、権限とログの設計も重くなる。

次に、顧客データの扱いを決める必要がある。税務・法務・監査・コンサルでは、顧客ごとの守秘義務、監査独立性、契約範囲、外部委託、国境を越えるデータ移転が絡む。AI が参照してよいデータ、生成してよい成果物、外部モデルへ渡してよい情報、保存期間を、ユースケースごとに分けるべきだ。

3つ目は、人間レビューの位置だ。AI エージェントが税務論点の整理、契約レビュー、脆弱性修正、IT 近代化のコード提案を行うとしても、最終判断を AI に置くべきではない。誰が専門判断を行い、どの出力を顧客に渡し、どの変更を本番に反映し、どの例外をエスカレーションするかを定義する必要がある。

4つ目は、効果測定の単位だ。AI を何人が使ったかではなく、業務サイクルがどう変わったかを見る。税制変更対応なら、情報収集から顧客説明までの時間。法務なら、契約ドラフトからレビュー完了までの差戻し回数。サイバーセキュリティなら、検出から修正適用までのリードタイム。PE 向けなら、投資先の IT 近代化や業務改善がどこまで短縮されたかを見る。

## まとめ

KPMG と Anthropic の提携は、Claude を 276,000 人超へ配る大規模導入であると同時に、AI エージェントを専門サービスの業務プラットフォームへ組み込む発表だ。Digital Gateway、Claude Cowork、Managed Agents、PE 向け導入支援、KPMG Blaze、サイバーセキュリティと AI assurance まで含めると、単なる生成 AI 利用拡大ではなく、顧客提供モデルの再設計として読むべきだ。

日本企業は、このニュースを「KPMG が Claude を採用した」で終わらせないほうがよい。自社で AI エージェントを業務基盤に入れるなら、データ境界、権限、人間レビュー、教育、監査ログ、顧客説明を先に設計する必要がある。専門サービスや規制業務で AI を使う競争は、モデル選定だけでなく、業務として安全に回せる設計力で決まる段階に入っている。
