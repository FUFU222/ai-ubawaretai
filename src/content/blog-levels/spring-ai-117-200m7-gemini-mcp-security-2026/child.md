Spring AI は、Java や Spring Boot で生成 AI アプリを作るためのフレームワークです。チャットモデル、ツール呼び出し、RAG、MCP 連携などを、Spring の作法で扱いやすくします。

2026年5月23日、Spring AI は 1.0.8、1.1.7、2.0.0-M7 を公開しました。初心者がまず見るべきなのは、1.1.7 にセキュリティ修正が入っていることです。2.0.0-M7 は次の大きな版に向けたお試し版なので、本番で急に使うものというより、将来の変更を確認するための版です。

## 何が起きたのか

今回の更新には、CVE-2026-41863 という脆弱性の修正が含まれます。CVE とは、広く共有される脆弱性番号です。Spring の説明では、Anthropic Skills API を扱う Spring AI の一部機能で、AI が関わったファイル名を十分に安全化しないまま、ファイル保存に使ってしまう可能性がありました。

簡単に言うと、AI が作ったファイル名をそのまま信用すると危ない、という話です。AI が「report.txt」と返すなら問題なさそうですが、もし保存してはいけない場所を指すような名前が混ざると、アプリが想定外の場所にファイルを書いてしまう可能性があります。

この問題は 1.1.7 以降で修正されています。もし Spring AI 1.0.0 から 1.1.6 を使っていて、Anthropic Skills API やファイル生成機能を使っているなら、早めに確認すべきです。

## 2.0.0-M7は何を見るものか

2.0.0-M7 には、MCP まわりの変更が入っています。MCP は、AI アプリが外部ツールやデータソースにつながるための仕組みです。たとえば、AI が社内ドキュメント、チケット、コード、API などを使うときの接続面になります。

今回のリリースでは、Streamable HTTP transport が既定になるなど、MCP の通信方法に関係する変更があります。初心者には少し難しく見えますが、要点は「AI と社内ツールをつなぐ道路の形が変わり始めている」ということです。

また、Gemini 2.5 Flash の更新や OpenAI の streaming 応答修正も含まれます。生成 AI アプリでは、モデルごとに応答の出方が違います。画面に少しずつ文章を表示する streaming も、モデルや API の違いで動きが変わることがあります。

## どの順番で対応すればよいか

まず、自分のプロジェクトが Spring AI を使っているか確認します。`pom.xml` や `build.gradle` に Spring AI の依存関係があるかを見ます。会社では、正式なサービスだけでなく、PoC や社内ツールにも入っているかもしれません。

次に、バージョンを見ます。1.0.0 から 1.1.6 を使っているなら、1.1.7 以降への更新を検討します。特に、AI がファイルを作る機能、Anthropic Skills API、Claude 連携を使っている場合は優先度が上がります。

そのうえで、2.0.0-M7 は検証環境で試します。本番にすぐ入れるのではなく、MCP、Gemini、OpenAI streaming、tool calling を使っている部分が動くかを確認します。AI アプリは「ビルドが通る」だけでは不十分で、実際の回答、ツール呼び出し、エラー時の動きも見る必要があります。

## まとめ

今回の Spring AI 更新は、Java の生成 AI アプリを作る人にとって大事なニュースです。1.1.7 はセキュリティ修正として確認し、2.0.0-M7 は次の変更を知るために検証します。

ポイントは、AI が作った値をそのまま信じないことです。ファイル名、ツール引数、保存先、API 呼び出しは、普通のユーザー入力と同じようにチェックします。生成 AI アプリでも、基本的なセキュリティ設計は変わりません。
