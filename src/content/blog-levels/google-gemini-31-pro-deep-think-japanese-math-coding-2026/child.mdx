Googleが2026年4月7日に出した公式ブログで、**Gemini 3.1 Pro** と **Gemini 3.1 Pro Deep Think** が、日本語の数学とコーディングでかなり強くなったことが発表された。今回のポイントは、ただ「AIが進化しました」と言っているだけではなく、日本の人にも分かりやすい **日本数学オリンピック（JMO）** や **ICPC横浜大会の国内予選** を使って説明しているところだ。

まず一番インパクトがあるのは、Deep ThinkがJMO本選で **100%** の最高得点を取ったという話だ。しかも、JMOの公式結果を見ると、2025年本選では受験者205人のうち満点の人はいなかった。人間の最高得点は80%だったので、Googleは「日本語の難しい数学問題でもかなり高い推論ができる」と見せたかったのだと思う。

もう一つ大事なのが、Deep Thinkだけでなく、普通の上位モデルである **Gemini 3.1 Pro** も強くなっていることだ。Googleによると、JMO本選ではGemini 3.1 Proが **70.0%**、前のGemini 3 Proは **47.5%** だった。つまり、特別モードだけがすごいのではなく、広く使われるモデル本体も大きく伸びている。

コーディングのほうでも、Deep ThinkはICPC横浜大会の国内予選で **97.8%** を出したという。ICPCは競技プログラミングの大会で、アルゴリズムを考えて正しいコードを書く力が問われる。日本の情報系の学生やプログラマーにはかなりイメージしやすい題材だ。

ここで大切なのは、「英語で強いAI」ではなく、**日本語の問題をそのまま理解して考えられるAI** だと示したことだと思う。日本の開発現場では、仕様書もエラーレポートもレビューコメントも日本語で動くことが多い。だから、日本語で考える力が弱いと、英語ベンチマークが高くても実際には使いにくい。

今回の発表が日本の開発者に効くのはそこだ。たとえば、日本語の要件を読ませて実装方針を考えさせる、日本語の不具合報告からテスト観点を出させる、日本語の説明を読んでアルゴリズムを組み立てる、といった仕事に相性がよくなる可能性がある。

教育の面でも意味がある。JMOやICPCが話題に出たことで、塾や大学や学習サービスが「日本語で数学や情報を説明できるAI」としてGeminiを見るきっかけになるかもしれない。答えを出すだけでなく、「なぜそう考えるのか」を日本語で説明できるなら、学びの道具としてかなり便利だからだ。

ただし、ここで勘違いしてはいけないこともある。数学オリンピックや競技プログラミングで強いからといって、すぐに現場の開発が全部うまくいくわけではない。実際の仕事には、古いコード、曖昧な仕様、チームのルール、セキュリティ、レビューなど、もっと複雑な条件があるからだ。

それでも、Googleが日本語の高度な問題でここまで具体的な数字を出した意味は大きい。最近Googleは、このサイトでも紹介した[Gemini APIのFlex/Priority](/blog/google-gemini-api-flex-priority-2026/)や[Gemini API Docs MCP](/blog/google-gemini-api-docs-mcp-agent-skills-2026/)など、開発者向けの仕組みをどんどん整えてきた。そこに「日本語でも推論が強い」という材料が加わったので、Geminiは日本市場でさらに存在感を強めそうだ。

Googleによると、Deep ThinkはGoogle AI Ultraユーザー向けにGeminiアプリで順次提供され、Gemini 3.1 ProはGemini API、Vertex AI、Gemini CLI、Android Studio、Geminiアプリ、NotebookLMなどへ広がっている。つまり、研究だけではなく、実際に使う場所へ落ち始めている。

今回のニュースを一言でいうなら、**Googleが「Geminiは日本語でも本気で使える」と数字付きで示してきた** ということだ。ここから先は、本当に日本の開発や教育の現場で同じ強さが出るのかが勝負になる。
