OpenAI は、ChatGPT のデスクトップアプリを更新し、Chat、Work、Codex を同じアプリの中で使いやすくしました。Chat はふつうの質問や会話、Work は調査や資料作成のような長い仕事、Codex はコードを書く作業やリポジトリを扱うための場所です。

大事なのは、見た目がまとまっても、すべてが一つに混ざったわけではないことです。Codex は別の画面として残り、Codex の作業履歴も ChatGPT の履歴とは分かれます。Work の会話は web、mobile、desktop で続けられるものがありますが、Codex は web や mobile から普通に選ぶ形ではありません。

## ChatGPT Classicが残ることがある

今まで ChatGPT のデスクトップアプリを使っていた人は、新しいアプリを入れた後も、古いアプリが ChatGPT Classic として残ることがあります。これはすぐに移行しない人のための扱いです。Classic も当面は更新やサポートを受けますが、新しい agent 系の機能は新しい ChatGPT アプリ側に寄っていく可能性があります。

会社では、ここを利用者に説明する必要があります。「ChatGPT が2つある」「どちらを使えばよいか分からない」という問い合わせが出やすいからです。ヘルプデスク向けには、どちらが新しいアプリか、Classic をいつまで残すか、どの部署が新しいアプリを使うかを決めておくとよいです。

## WorkとCodexは役割が違う

Work は、調べもの、情報整理、文書、表計算、レポート、Sites のような成果物を作る作業に向いています。Codex は、ローカルファイル、リポジトリ、ターミナル、開発ツールとつながる開発作業に向いています。

この違いは、会社のルールにも関係します。[ChatGPT agentからWorkへの移行](/blog/openai-chatgpt-agent-work-migration-2026/)で見たように、長い業務作業は Work に寄っていきます。一方、開発作業は Codex として管理したほうが分かりやすいです。同じ ChatGPT アプリの中にあっても、許可する人、扱ってよいデータ、確認するログは分けて考える必要があります。

また、[Atlas終了とブラウザ作業の移行](/blog/openai-atlas-retirement-browser-agent-migration-2026/)のように、OpenAI はブラウザやデスクトップで行う AI 作業を整理し直しています。便利になったから全員に同じ権限を渡すのではなく、業務と開発で使い方を分けることが大切です。

## 管理設定が勝手に変わるわけではない

OpenAI は、今回のナビゲーション更新で、既存の workspace access、security settings、governance controls、spend model は変わらないと説明しています。つまり、新しいアプリになったからといって、会社の管理設定が自動的に緩くなるわけではありません。

ただし、利用者から見ると「新しい機能が使えるようになった」と感じやすいです。Work が見える人と見えない人がいたり、Codex が別画面にあったり、Classic が残ったりすると、混乱しやすくなります。そのため、会社はアプリの配布だけでなく、使い分けの説明を用意する必要があります。

## 会社が確認すること

まず、端末にどのアプリが入っているかを確認します。新しい ChatGPT app、ChatGPT Classic、Codex app が混ざっていないかを見ます。次に、どの部署に Work を使わせるか、どの部署に Codex を使わせるかを決めます。

次に、FAQ を作ります。「Work が見えない」「Codex の履歴が見つからない」「Classic と新アプリの違いは何か」「Project はどこで使えるか」という質問に答えられるようにします。[ChatGPTのProject filesと統合検索](/blog/openai-chatgpt-unified-search-project-files-2026/)のように、便利な導線が増えるほど、どこに情報があるかの説明が重要になります。

最後に、権限を棚卸しします。Work、Codex、Plugins、Sites、Computer Use、browser use などは、それぞれできることが違います。新しいデスクトップアプリを入れる前に、誰が何を使えるのかを確認しておくと、あとから慌てずに済みます。

## まとめ

ChatGPT の新しいデスクトップアプリは、Chat、Work、Codex を同じ入口に近づける更新です。ただし、Codex の履歴は分かれ、ChatGPT Classic が残ることもあり、Work の展開にも差があります。

会社で使うときは、「アプリが一つになった」とだけ説明しないほうがよいです。何が一つになり、何が別のままなのかを伝える。これが、利用者の混乱を減らし、安全に使い始めるための第一歩です。
