Anthropic は、Claude Opus 4.1 を古いモデルとして扱い、2026年8月5日に使えなくする予定だと発表しました。代わりに使う候補は Claude Opus 4.8 です。

これは「新しいAIが出た」という話だけではありません。会社のアプリや開発ツールがまだ Opus 4.1 を呼んでいる場合、そのままだと期限後に動かなくなる可能性があります。

## 何が起きたの？

2026年6月5日の Anthropic のリリースノートでは、Claude Opus 4.1 が deprecated になったと説明されています。deprecated とは、「これから新しく使うのはやめて、移行してください」という合図です。

さらに、Anthropic のモデル廃止ページでは、direct Anthropic API での Opus 4.1 retirement が 2026年8月5日とされています。retirement は、実際にそのモデルを呼べなくなる期限と考えると分かりやすいです。

## なぜ会社に関係するの？

会社でAIを使うとき、モデル名は意外な場所に残ります。アプリのコード、環境変数、CI、社内のAIゲートウェイ、Notebook、テスト用スクリプト、古い手順書などです。

もしそこに Opus 4.1 が残っていると、8月5日以降に処理が失敗するかもしれません。チャットで少し使うだけなら気づきやすいですが、夜間バッチ、社内検索、コードレビュー支援、問い合わせ分類のような裏側の処理では、原因が分かりにくくなります。

## どう移行すればいいの？

まず、Opus 4.1 をどこで使っているか探します。コード検索だけでなく、設定ファイル、CI、クラウド設定、社内ドキュメントも見ます。`Opus 4.1` という文字だけでなく、社内で付けた別名も探す必要があります。

次に、Opus 4.8 へ変えたときに出力がどう変わるかを試します。新しいモデルのほうがよくなることもありますが、文章の長さ、コードの直し方、説明の仕方が変わることもあります。大事な仕事では、いきなり本番に入れず、小さなテストから始めます。

最後に、戻し先を決めます。8月5日を過ぎると、Opus 4.1 に戻すことは前提にできません。問題が出たときに、別の現行モデルへ戻すのか、プロンプトを調整するのか、人間確認に切り替えるのかを決めておきます。

## クラウド経由にも注意

Claude を Anthropic から直接使っている会社もあれば、AWS Bedrock や Google Vertex AI などを通して使っている会社もあります。経路によって、使えるモデルや終了日が違う場合があります。

そのため、管理者は「うちはどこからClaudeを呼んでいるか」を先に確認する必要があります。開発チーム、AI推進チーム、情シス、クラウド管理者が別々に見ていると、どこかに古い設定が残りやすいからです。

## まとめ

Claude Opus 4.1 の廃止は、ただのニュースではなく、期限付きの移行作業です。2026年8月5日までに、どこで使っているかを探し、Opus 4.8 などの現行モデルで試し、問題が出たときの戻し方を決める必要があります。

会社でAIを安全に使うには、新しいモデルを試すだけでなく、古いモデルを確実に片づけることも大切です。
