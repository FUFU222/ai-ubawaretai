Anthropic は 2026年7月9日、Claude の使い方を振り返る **Reflect** という beta 機能を出しました。Claude の Settings から開くと、自分がどんな話題で Claude を使っているか、どの時間帯に使っているか、どんな作業を任せがちかを見られます。

これは、AIの利用量を増やすためだけの機能ではありません。自分がAIをどう使っているのかを見直し、どの作業はAIに任せ、どの作業は自分で考えるべきかを考えるための機能です。

## 何が見えるのか

Reflect では、過去 1か月、3か月、6か月、12か月の Claude chat activity を振り返れます。よく話したテーマ、使った時間帯、作業の種類などがまとめられます。

たとえば、文章の下書きが多いのか、調査が多いのか、仕事の整理が多いのかを見られます。Anthropic は、AIをどのくらい使うべきか、AIが向いている作業は何か、人間が自分でやるべき作業は何かを考えるために作ったと説明しています。

Claude は、利用者に「Claudeが速くできても、自分で続けたいことは何か」のような問いを出すこともあります。これは、AIに何でも任せるのではなく、自分の考え方や仕事の進め方を守るための問いです。

## メモリが必要になる

Reflect は、Claude の memory がオンになっている人向けの機能です。Free、Pro、Max plan の web と Claude Desktop で beta 提供されています。

ここが大事です。Reflect は、ただのアクセス回数カウンターではありません。Claude が過去のやり取りから利用パターンをまとめるため、メモリと関係します。便利な一方で、どんな会話を業務アカウントでするかを考える必要があります。

Anthropic は、incognito chats は Reflect に使わないと説明しています。また、接続ツールの元ファイルそのものを取り込むわけではありません。たとえばメールを要約してもらった場合、元メールそのものではなく、Claude との会話の傾向が出る可能性があります。

健康連携ツールにつながった会話は insights から除外されます。それでも、仕事、悩み、生活、健康、家計、人間関係のような話題は、まとめ方によっては本人にとって敏感な情報になります。

## 休むための設定もある

Reflect とあわせて、Settings > Time and focus では quiet hours や break reminders を設定できます。quiet hours は Claude を使わない時間帯を決めるための設定です。break reminders は、長く使ったときに休憩を促すための設定です。

AIは便利なので、つい長く使ってしまいます。調査、文章作成、相談、コードレビュー、資料作成を全部AIに聞いていると、自分で判断する時間が減ることもあります。だから、使う時間を見直す機能には意味があります。

日本企業でAIを導入するときも、利用率だけを見ればよいわけではありません。たくさん使っていることが、いつも良いこととは限りません。大切なのは、AIを使って仕事の質が上がっているか、人間が判断すべきところを残せているかです。

## 会社ではどう使うべきか

会社で Reflect を扱うなら、社員を監視するための機能として説明しないほうがよいです。今回の Reflect は、基本的に利用者本人が自分の使い方を見るためのものです。

むしろ、AI研修に使いやすい機能です。たとえば、毎月1回、自分の Reflect を見て「AIに任せてよかった作業」「自分で判断すべきだった作業」「メモリに残したくない内容」を振り返る、といった使い方ができます。

業務用アカウントでは、個人の悩み、健康、家族、転職、金銭、労務相談のような内容をどこまで扱ってよいかも決める必要があります。Claude が元ファイルを直接使わないとしても、会話の中に書いた情報は別です。

## 覚えておきたいこと

Claude Reflect は、自分のAI利用を見直すためのダッシュボードです。使った量だけでなく、どんな作業を任せているかを考えられる点が重要です。

日本企業では、Reflect を「AIをもっと使わせる機能」としてではなく、「AIと人間の役割分担を見直す機能」として扱うべきです。メモリ、incognito、quiet hours、break reminders を理解して使えば、AIをより安全に仕事へ定着させやすくなります。
