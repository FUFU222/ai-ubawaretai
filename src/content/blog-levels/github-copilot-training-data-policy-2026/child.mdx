GitHub Copilot の新しい方針で、「Copilot を使ったときのやり取りが学習に使われることがある」という点が大きな話題になりました。

しかも、初めからオフではなく、自分で設定を変えないといけない形だと受け取られたため、開発者の不安が強くなりました。

## 何が気になるの？

開発者は、AI の答えが便利かどうかだけでなく、「自分たちの情報がどう扱われるか」もかなり気にしています。

コードそのものだけでなく、質問のしかた、問題の相談内容、貼り付けた断片などにも、会社の事情がにじむことがあるからです。

だから「学習に使うかどうか」は、かなり大きな話です。

## private repo なら安心？

そこが分かりにくいところです。

企業向けの契約や private repo の扱いには例外や条件があることも多いですが、普通の利用者から見ると、設定画面や説明が分かりにくいと不安が残ります。

大事なのは、「たぶん大丈夫」で終わらせず、自分の契約や設定を確認することです。

## なぜ反発が起きるの？

AI コーディングツールは、日々の開発に深く入り込みます。

だからこそ、後から「その会話も学習に使うかもしれません」と言われると、信頼が揺れやすいです。

便利さと引き換えに何を渡しているのかが、前よりずっと大事になっています。

## 使う側はどうすればいい？

まず、自分の設定を確認すること。

次に、会社で使うなら、組織として opt-out や利用ルールを決めること。

そして、機密情報や未公開情報をどこまで入れてよいかを、あいまいにしないことです。

## まとめ

今回の話は、Copilot が便利かどうかより、「便利な道具をどこまで信用してよいか」という話です。

AI ツールを仕事に入れるなら、性能だけでなく、データの扱いも同じくらい見ないといけません。
