OpenAI は 2026年6月17日に、ChatGPT の Scheduled tasks を使いやすくする更新を発表しました。Scheduled tasks は、ChatGPT に「あとで知らせて」「毎朝まとめて」「変化があったら教えて」と頼める機能です。

今回の大きな変更は、タスクを管理する場所が分かりやすくなったことです。新しい Scheduled page から、今動いているタスク、次に動く時間、停止、再開、編集、削除を確認できます。以前より、作ったタスクを探しやすくなります。

## 何が変わったのか

まず、特定の時刻だけでなく、朝、午後、夕方のような広い時間帯で予定を作りやすくなりました。たとえば「毎朝、今日の重要ニュースをまとめて」「毎週金曜の午後に、来週の準備を思い出させて」のような使い方です。

次に、監視タスクがあります。これは ChatGPT が Web や接続されたアプリを確認し、意味のある変化があったときだけ知らせる使い方です。ただし、これは本格的なシステム監視ではありません。重要な障害、契約期限、法務判断などを ChatGPT の通知だけに任せるのは危険です。

OpenAI は、Pulse という proactive update 機能を終了し、こうした更新を Scheduled tasks に寄せる流れも示しました。つまり、ChatGPT は会話するだけでなく、あとで動く道具にも近づいています。

## 使う前に知るべき制限

Scheduled tasks には上限があります。プランによって持てるアクティブタスク数が違い、Plus は 5 件、Business と Edu は 10 件、Pro と Enterprise は 15 件と説明されています。また、タスクは 1時間に1回を超えて実行できません。

タスクは放置されると自動で止まることがあります。関連するチャットを削除した場合も、タスクは止まります。つまり、ChatGPT のタスクは便利な補助機能であり、会社の重要な定期処理を確実に動かすジョブ基盤ではありません。

使える場所にも違いがあります。Web とモバイルでは Scheduled page を使えますが、現時点では ChatGPT desktop app や Codex app の Scheduled tab ではありません。会社で案内するときは、どの画面から作るのかを明確にしたほうがよいです。

## 会社で気をつけること

会社で使う場合、最初は小さく始めるのが安全です。たとえば、社内FAQの更新確認、会議前のリマインダー、週次ニュースのまとめ、公開ページの変更確認などです。これらは、通知が間違っても人間が確認できます。

一方、メール、ファイル、カレンダー、社内SaaSとつなぐと注意が必要です。ChatGPT がアプリを使うには権限が関係します。Business や Enterprise では、管理者がどのアプリを使えるか決める場合があります。

また、ChatGPT agent や Codex automations と混同しないことも大事です。Scheduled tasks は、ChatGPT でリマインダーや定期チェックをする機能です。ChatGPT agent は Web 操作やファイル処理など、もっと広い作業を進める機能です。Codex automations は開発作業向けの自動化です。

## まとめ

ChatGPT の Scheduled tasks 更新は、毎日の小さな確認や通知を楽にする機能です。ただし、会社で使うなら「何を任せてよいか」「どのアプリを使ってよいか」「誰が止めるか」を決める必要があります。

最初は、低リスクな通知から使うのがよいです。重要な判断や顧客情報を扱う仕事では、人間の確認、管理者設定、アプリ権限、停止手順を必ずセットにしましょう。
