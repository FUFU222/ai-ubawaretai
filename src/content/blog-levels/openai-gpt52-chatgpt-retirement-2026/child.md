OpenAI は 2026年6月12日、ChatGPT で GPT-5.2 models を退役させました。対象は GPT-5.2 Instant、GPT-5.2 Thinking、GPT-5.2 Pro です。過去の会話は消えませんが、続きを書くときは対応する GPT-5.5 モデルで続くと説明されています。

これは、新しい機能の発表というより、ChatGPT で使えるモデルを整理する更新です。会社で ChatGPT を使っている場合、古いモデル名を前提にした手順書や研修資料が残っていないか確認する必要があります。

## 何が変わったのか

GPT-5.2 系は ChatGPT のモデル選択から外れました。以前 GPT-5.2 で始めた会話も、今後は GPT-5.5 系で続きます。つまり、同じ会話でも、今後の返答の雰囲気や長さが少し変わる可能性があります。

OpenAI は、後継モデルが出たあと、古いモデルを ChatGPT でおおむね 90 日残す方針を示しています。今回の退役も、その流れの一部です。今後もモデル名は変わるものとして考えたほうがよいです。

ただし、これは ChatGPT の話です。API を使っているシステムが、この発表だけで自動的に変わるわけではありません。ChatGPT の画面利用と API 利用は分けて考える必要があります。

## 会社で確認するもの

まず、社内資料から `GPT-5.2` という言葉を検索します。Notion、Google Drive、Slack、社内 wiki、研修資料、プロンプト集などに残っているかもしれません。

次に、長く続けている ChatGPT の会話を確認します。企画、法務、営業、採用、開発設計などで、同じ会話を何週間も使っている場合、続きを GPT-5.5 系で書いたときに違和感がないか見ます。

GPTs も確認します。社内で共有している GPTs の説明に「GPT-5.2 推奨」と書かれているなら、説明を直す必要があります。誰が管理する GPTs なのかも決めておくと、次のモデル変更のときに困りにくくなります。

## 開発チームの注意点

開発チームは、ChatGPT で試した結果と API の本番挙動を混ぜないことが大切です。ChatGPT 側のモデルが GPT-5.5 に変わっても、API のモデル指定は別に管理します。

プロンプトを検証するときは、使ったモデル名と日付を残します。あとで出力が変わったときに、モデルが変わったのか、プロンプトが悪いのかを切り分けやすくなります。

重要な業務では、固定モデル、評価データ、ロールバック方法を用意します。最新モデルを試す環境と、本番で安定して使う環境を分けると安全です。

## まとめ

GPT-5.2 の退役は、ChatGPT を仕事で使う会社にとってモデル棚卸しの合図です。古い会話は続けられますが、今後は GPT-5.5 系で動きます。

日本企業は、社内資料、長期会話、GPTs、プロンプト集、研修資料を確認し、古いモデル名に依存していないか見直すべきです。API については、ChatGPT の変更と混同せず、別に管理することが重要です。
