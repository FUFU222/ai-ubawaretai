Gemini API Docs MCPとDeveloper Skillsの発表は、一見すると小粒な開発者向け改善に見える。しかし実際には、**コーディングエージェントの信頼性をどこで担保するか**という問題に対する、かなり構造的な答えだ。

いまのコーディングエージェントは、推論能力そのものより、**参照している知識の鮮度と信頼境界**で失敗することが多い。コード生成モデルは十分それらしいコードを書けるが、そのコードが最新SDKに合っているか、推奨パターンに沿っているか、今の制約条件を踏まえているかは別問題だ。

## 「学習済みの強いモデル」だけでは足りない

ここ数か月の開発現場では、AIが壊れたコードを出す理由として次の3つが目立つ。

- 学習時点のAPI知識が古い
- 新しいSDKパターンへ追随できていない
- 公式ドキュメントより、古いサンプルや断片的な記憶を優先してしまう

この問題に対して、「より大きなモデルなら解決する」と考えるのは甘い。むしろ必要なのは、**モデルの外にある最新文脈を安定供給する仕組み**だ。Docs MCPはそこを狙っている。

MCPの利点は、単に情報取得できることではない。**どの情報源を、どのプロトコルで、どの境界の中で渡すかを明示できる**ことにある。雑なWeb検索ベースのRAGでは、検索品質、取得先の信頼性、情報の古さ、ノイズ混入の問題が残る。Docs MCPは、その不確実性をかなり減らせる。

## Developer Skillsの意味

一方で、最新ドキュメントへつながるだけでは不十分だ。ドキュメントは広く、エージェントは必ずしも最短経路で問題を解かない。そこでDeveloper Skillsが効く。

Skillsは、単なる補助テキストではなく、**検索空間を狭めるための運用知識**だ。どの資料を優先するか、どういうSDKパターンを選ぶか、何を避けるか。これを持つことで、エージェントは同じ情報源を見ていても、より現行のベストプラクティスへ寄りやすくなる。

要するに、Docs MCPが**情報の鮮度**を担い、Developer Skillsが**行動のバイアス**を担っている。この分業はかなり理にかなっている。

## 数字の読み方

GoogleはMCPとSkillsの併用で、96.3%のpass rateと63%少ないトークンを報告している。これは社内評価なので、そのまま一般化はしにくい。ただし重要なのは数字の絶対値より、**改善の方向が自然であること**だ。

正しいドキュメントへ早く到達し、余計な試行錯誤を減らせば、トークン消費は減り、成功率は上がる。つまりこの結果は、「より賢い思考」ではなく、**より良い文脈供給**が効いていることを示している。

ここは今後のAI開発でかなり重要になる。LLMの性能向上が頭打ちになるとは思わないが、開発体験を左右するボトルネックは、モデル本体より**文脈供給の品質**へ移りつつあるからだ。

## これはGeminiだけの話ではない

もっと広く見ると、この発表は「APIベンダーは、SDKとドキュメントをAI可読な形で配布すべき」という流れの始まりに見える。

今後は公式ドキュメントが単にHTMLで存在するだけでは弱い。AIが読む前提で、

- 最新性が明示されている
- 参照先が管理されている
- 実装パターンが構造化されている
- ベストプラクティスがスキルとして渡せる

といった条件が重要になる。

つまりドキュメントは、マーケティング資産でもサポート資産でもなく、**エージェント向け実行インフラの一部**になる。Docs MCPは、その方向をかなり分かりやすく体現している。

## 実務面での含意

開発チーム側でも、この変化に合わせて考え方を変える必要がある。これからは、コーディングエージェントへ何をさせるか以上に、**どの情報源を読ませるか**が重要になる。

社内ライブラリや社内APIでも同じだ。人間向けWikiに散らばった説明だけでは、エージェントが安定して使えない。構造化されたドキュメント、更新されたサンプル、推奨パターン集、アクセス制御された文脈配布が必要になる。

その意味で、今回のGoogleの発表は、単なるGemini API支援ではなく、**エージェント時代の開発者プラットフォーム設計**そのものに踏み込んでいる。

## まとめ

Docs MCPとDeveloper Skillsの価値は、コーディングエージェントに「もっと考えさせる」ことではなく、**正しい現在地へ接続すること**にある。コーディングAIの信頼性問題は、モデル単独の問題ではなく、文脈の供給設計の問題でもある。

Googleはそこに対して、公式ドキュメントのMCP配布とベストプラクティスのスキル化という、かなり再利用性の高い答えを出してきた。今後は他社も、似た形でドキュメントとSDKをAI可読化していく可能性が高い。
